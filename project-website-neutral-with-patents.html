Proximity-Gated Biometric Access Controller

Two-factor physical access that works without cloud

A registered smartphone must first pass BLE cryptographic attestation; only then is the fingerprint sensor enabled. Final decision is made locally/offline, with anti-relay latency windows, secure element, and tamper-evident (hash-chained) logs. Interfaces: OSDP Secure Channel / Wiegand, relays, CAN/LIN.

Key capabilities

  • BLE attestation (ECDH/AES-GCM), signed policies, nonces/counters
  • Biometric PAD; encrypted local templates; privacy-by-design
  • Anti-relay timing windows; fail-secure + offline continuity
  • Tamper sensors + hash-chained audit; SIEM export
  • OSDP/Wiegand + relays/dry-contact; CAN/LIN for elevators/lockers/ECUs
Latency P50/P95
FAR/FRR + PAD
Offline ≥72h
Tamper ≤5s alert

Problem

Badges/PINs are easily cloned; many mobile readers don’t prove proximity or fail securely. Critical facilities need cloud-independent access and audit.

Solution

Enforce phone-then-fingerprint. Decision stays at the edge; logs are tamper-evident; works through outages; broad I/O for doors, racks, lockers, elevators.

Market Fit

Telecom, datacenters, logistics/industrial, healthcare/labs, and fleet. Integrates with existing PACS via OSDP/Wiegand; extends to CAN/LIN systems.

Pilot outline (example)

  • Sites: 1–2 base-station shelters + 1 POP/DC room (or doors/racks/lockers/elevator enable at an enterprise site)
  • Integrations: OSDP/Wiegand panel, relays; optional CAN/LIN; SIEM feed
  • KPIs: latency P50/P95, FAR/FRR (PAD), offline endurance, anti-relay rejection, tamper-to-alert
